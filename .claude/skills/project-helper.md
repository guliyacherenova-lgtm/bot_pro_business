# Скилл проекта: AI-бот квалификации кандидатов NL International

Этот скилл содержит специализированные знания и инструкции по поддержке и развитию бота для первичного отбора кандидатов в NL International.

## 🗺 Карта проекта
- `main.py`: Основная логика, обработчики Telegram, состояния FSM (машина состояний) и интеграция с GigaChat.
- `config.py`: Управление конфигурацией и загрузка переменных окружения.
- `.env.example`: Шаблон необходимых переменных окружения.
- `requirements.txt`: Список зависимостей проекта.
- `README.md`: Публичная документация проекта и скриншоты.
- `PROJECT_DESCRIPTION.md`: Детальные бизнес-цели и описание целевой аудитории.
- `images/`: Скриншоты проекта для документации.

## 🚀 Инструкции по эксплуатации

### 1. Запуск проекта
Для запуска бота:
- Убедитесь, что файл `.env` настроен и содержит `BOT_TOKEN`, `GIGACHAT_CLIENT_ID`, `GIGACHAT_CLIENT_SECRET` и `REF_LINK`.
- Выполните команду: `python main.py`

### 2. Внесение изменений
При изменении логики бота:
- **Обработчики (Handlers)**: Найдите нужное состояние в FSM (Finite State Machine) внутри файла `main.py`.
- **Промпты**: Промпты для GigaChat находятся в функциях обработки AI. Всегда тестируйте изменения промптов на разных вариантах ответов пользователей.
- **Конфигурация**: Новые настройки добавляйте в `config.py` и обязательно обновляйте `.env.example`.

### 3. Обновление документации (README)
Стандартный процесс обновления README:
1. **Захват**: Сделайте новые скриншоты работы бота.
2. **Конвертация**: Переведите изображения в форматы `.png` или `.jpg`.
3. **Загрузка**: Поместите изображения в папку `images/`.
4. **Ссылка**: Обновите ссылки в `README.md`, используя формат `![Описание](images/название_файла.png)`.

### 4. Поиск ошибок и отладка
- **Отсутствующие ключи**: Если бот падает при старте, проверьте, все ли переменные из `.env.example` перенесены в `.env`.
- **Ошибки AI**: Если GigaChat выдает странные ответы, проверьте логи консоли на наличие ошибок API или таймаутов.
- **Зависание диалога**: Если пользователь «застрял» в диалоге, проверьте, что все возможные вводы (текст/кнопки) обработаны для этого конкретного состояния.
