# AI Operational Skill: NL-Bot-Developer

  Этот скилл определяет стандартные операционные процедуры (SOP) для Claude
  Code при работе с репозиторием бота квалификации NL International.

  ## 🛠 Техническая архитектура (Project Map)
  - **Логика**: `main.py` (обработчики aiogram 3.x, FSM-состояния,
  интеграция GigaChat).
  - **Конфигурация**: `config.py` (загрузка через pydantic/dotenv).
  - **Окружение**: `.env` (секреты), `.env.example` (шаблон).
  - **Зависимости**: `requirements.txt`.
  - **Документация**: `README.md` (публичная), `PROJECT_DESCRIPTION.md`
  (бизнес-логика).

  ## 🤖 Алгоритмы действий для Claude Code

  ### 1. Запуск и проверка сервиса
  При запросе на запуск или проверку:
  - Проверить наличие `.env` и соответствие его полей `.env.example`.
  - Запустить команду: `python main.py`.
  - Мониторить консоль на наличие `aiogram` ошибок или таймаутов GigaChat
  API.

  ### 2. Внесение изменений в код
  При модификации функционала:
  - **FSM**: Изменения в сценарии диалога вносятся через обновление
  состояний в `main.py`.
  - **AI-Prompts**: Обновление промптов для GigaChat требует обязательного
  тестирования на 3+ вариантах пользовательского ввода.
  - **Config**: Любая новая переменная должна быть добавлена в `config.py` и
  продублирована в `.env.example`.

  ### 3. Автоматизация обновления README (SOP)
  Если пользователь просит обновить документацию или добавить скриншоты:
  1. Запросить у пользователя актуальные изображения.
  2. Проверить формат: если `.heic`, выполнить конвертацию в `.png` или
  `.jpg`.
  3. Переместить файлы в директорию `images/`.
  4. Обновить Markdown-разметку в `README.md` по шаблону:
  `![Описание](images/filename.png)`.

  ### 4. Диагностика и исправление ошибок
  При возникновении багов:
  - Проверить логи на `ModuleNotFoundError` $\rightarrow$ выполнить `pip
  install -r requirements.txt`.
  - Проверить `Invalid token` $\rightarrow$ указать пользователю проверить
  `.env`.
  - Проверить «зависания» бота $\rightarrow$ проанализировать переходы между
  состояниями FSM в `main.py`.

  ## 📦 Регламент подготовки к публикации
  Перед финальным коммитом / публикацией выполнить проверку:
  - [ ] **Security**: Убедиться, что `.env` исключен из списка загружаемых
  файлов.
  - [ ] **Clean-up**: Удалить или игнорировать `__pycache__` и `.vscode`.
  - [ ] **Sync**: Соответствие `README.md` актуальному функционалу кода.
  - [ ] **Assets**: Все изображения в `README.md` доступны и открываются.
  - [ ] **Deps**: Актуальность `requirements.txt`.
